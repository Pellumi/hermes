Data Architecture & Telemetry Model
Autonomous Drone Operating System (ADOS)


Version: 1.0
Scope: Simulation + Supervised Autonomy (Abuja Pilot)
Status: Final - Approved for Implementation


1. Purpose
This document defines: 
* System-wide data domains and boundaries
* Complete telemetry schema with all operational fields
* Mission schema with contact and verification tracking
* Drone health and battery degradation models
* Contact operations schema
* Visual identification schema
* Weight verification schema
* Rerouting decision schema
* Warehouse docking management
* Geofence violation tracking
* Event taxonomy (50+ events covering full mission lifecycle)
* Storage strategy with TimescaleDB optimization
* Retention policies with graduated compression
* Data integrity, security, and audit requirements
* Regulatory compliance (NCAA, NITDA, GDPR)
It ensures: 
* Autonomy models are trainable from real telemetry
* State transitions are fully auditable
* Regulatory compliance is achievable (NCAA Part 18)
* Scaling from 1 to 1000+ drones without redesign


2. Architectural Data Philosophy
ADOS data is divided into three classes:
Class
	Purpose
	Examples
	Operational
	Real-time mission execution
	Telemetry, state transitions, commands
	Analytical
	Model training & optimization
	Aggregated telemetry, reroute patterns, battery degradation
	Regulatory/Audit
	Compliance & Investigations
	Immutable logs, operator actions, incident records
	

Guiding Principles:
* Append-only logs for safety and state events (no deletions)
* Event-driven propagation (all state changes emit events)
* Time-series optimized telemetry storage (TimescaleDB)
* Geospatial-aware routing and tracking (PostGIS)
* Explicit separation of control-plane vs data-plane
* HMAC-signed telemetry with replay protection
* PII classification for GDPR compliance
* Immutable audit trail for regulatory investigations

 timescaledb.compress,

4.3 Data Flow Architecture
Drone/Sensor → MQTT (TLS) → Telemetry Ingestion Service
                              ├──–> Validation Layer (schema check)
                              ├──–> Dead Letter Queue (on failure)
                              ├──–> TimescaleDB (persist)
                              ├──–> Redis (live state cache)
                              ├─—-> WebSocket (operator dashboard)
                              └──–> Kafka (external webhooks)


Critical Components:
* Validation Layer: Rejects malformed telemetry, logs to DLQ
* Dead Letter Queue: Failed messages stored for replay (retention: 7 days)
* Monitoring: Latency budget <2s end-to-end (per System Architecture 7.2)


5. Telemetry Schema (Core Data Plane)
Telemetry emitted at minimum 1Hz, 5Hz during emergencies


5.1 Table: drone_telemetry
CREATE TABLE drone_telemetry (
   -- Identifiers
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   mission_id UUID NOT NULL,
   drone_id TEXT NOT NULL,
   sequence_number BIGINT NOT NULL,  -- Monotonic per drone (replay protection)
   timestamp TIMESTAMPTZ NOT NULL,
   
   -- Position (PostGIS)
   location GEOGRAPHY(POINT) NOT NULL,
   altitude_msl FLOAT,  -- meters
   altitude_agl FLOAT,  -- meters
   gps_accuracy_m FLOAT,
   gps_satellites INTEGER,
   
   -- Movement
   ground_speed_mps FLOAT,
   vertical_speed_mps FLOAT,
   heading_deg FLOAT,
   
   -- Battery & Power
   battery_percent FLOAT CHECK (battery_percent BETWEEN 0 AND 100),
   battery_voltage FLOAT,
   battery_current FLOAT,
   battery_temperature_c FLOAT,
   power_consumption_w FLOAT GENERATED ALWAYS AS (battery_voltage * battery_current) STORED,
   
   -- Payload Container (Autonomy Definition Section 19)
   payload_weight_kg FLOAT,
   container_status TEXT CHECK (container_status IN (
       'STOWED', 'LOWERING', 'LOADING', 'SEALED', 
       'DEPLOYING', 'EMPTY', 'FAULT'
   )),
   container_lock_status TEXT CHECK (container_lock_status IN (
       'LOCKED', 'UNLOCKED', 'FAULT', 'CALIBRATING'
   )),
   container_temp_c FLOAT,
   container_humidity FLOAT,
   container_cycle_count INTEGER DEFAULT 0,
   winch_current_ma FLOAT,
   winch_status TEXT CHECK (winch_status IN (
       'STOWED', 'LOWERING', 'RAISING', 'STALL', 'FAULT'
   )),
   
   -- Local Command Tracking (Autonomy Definition Section 12.1)
   local_command_received BOOLEAN DEFAULT FALSE,
   local_command_type TEXT CHECK (local_command_type IN (
       'VERIFIED_LOCAL', 'REJECTED_LOCAL', 'NONE'
   )),
   local_command_latency_ms INTEGER,  -- Time from container to drone
   
   -- Communication (Mission State Machine Section 4.3, 4.5)
   contact_attempts INTEGER DEFAULT 0,
   contact_mode_active TEXT CHECK (contact_mode_active IN (
       'PHONE', 'WHATSAPP_CALL', 'WHATSAPP_MSG', 'SMS', 'NONE'
   )),
   contact_established BOOLEAN DEFAULT FALSE,
   contact_latency_s FLOAT,  -- Seconds to establish
   signal_strength_dbm INTEGER,  -- Actual RSSI, not 0-5 scale
   signal_quality_percent FLOAT,  -- 0-100
   
   -- Visual Identification (Future Phase 3)
   camera_status TEXT CHECK (camera_status IN ('ACTIVE', 'STANDBY', 'FAULT')),
   visual_confidence FLOAT CHECK (visual_confidence BETWEEN 0 AND 1),
   visual_escalated BOOLEAN DEFAULT FALSE,
   visual_processing_ms INTEGER,
   
   -- Environment
   wind_speed_mps FLOAT,
   wind_direction_deg FLOAT,
   temperature_c FLOAT,
   
   -- Safety & Geofence
   geofence_distance_m FLOAT,  -- Distance to nearest no-fly boundary
   geofence_violation BOOLEAN DEFAULT FALSE,
   operator_intervention_active BOOLEAN DEFAULT FALSE,
   operator_id UUID NULL,  -- If intervention active
   
   -- Mission Context
   state TEXT NOT NULL,  -- Current mission state (per state machine)
   reroute_count INTEGER DEFAULT 0,
   fault_flags TEXT[] DEFAULT '{}',  -- Array of active fault codes
   
   -- Signatures (Security)
   hmac_signature TEXT NOT NULL,  -- HMAC-SHA256 of all fields
   client_timestamp TIMESTAMPTZ NOT NULL,  -- Original drone timestamp
   
   -- Indexes (defined below)
   PRIMARY KEY (mission_id, timestamp DESC)
);

-- Required Indexes
CREATE INDEX idx_telemetry_drone_time ON drone_telemetry (drone_id, timestamp DESC);
CREATE INDEX idx_telemetry_state ON drone_telemetry (mission_id, state);
CREATE INDEX idx_telemetry_battery ON drone_telemetry (drone_id, battery_percent) WHERE battery_percent < 30;
CREATE INDEX idx_telemetry_faults ON drone_telemetry USING GIN (fault_flags);
CREATE INDEX idx_telemetry_location ON drone_telemetry USING GIST (location);

-- Partial index for active emergencies
CREATE INDEX idx_telemetry_emergency ON drone_telemetry (timestamp) 
   WHERE 'EMERGENCY' = ANY(fault_flags);
	

5.2 Telemetry Validation Rules
Field
	Validation
	Action on Failure
	timestamp
	Must be within 60s of server time
	Reject (replay protection)
	sequence_number
	Must be > last sequence for drone
	Reject (replay protection)
	hmac_signature
	Must match computed HMAC
	Reject (tampering detection)
	battery_percent
	0-100 range
	Clamp, log warning
	location
	Within Abuja bounding box
	Accept, flag for review
	state
	Must be a valid mission state
	Reject, operator alert
	

6. Mission Schema (Control Plane)
6.1 Table: missions
CREATE TABLE missions (
   mission_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   drone_id TEXT NOT NULL,
   business_id UUID NOT NULL,
   mission_type TEXT CHECK (mission_type IN (
       'BUSINESS_TO_CUSTOMER', 'PEER_TO_PEER', 'API_INTEGRATION'
   )),
   mission_profile JSONB,  -- Priority, config overrides
   
   -- Waypoints (PostGIS)
   pickup_waypoint GEOGRAPHY(POINT) NOT NULL,
   destination_waypoint GEOGRAPHY(POINT) NOT NULL,
   return_waypoint GEOGRAPHY(POINT),  -- Warehouse or business
   
   -- Contact Information (Autonomy Definition Section 20)
   sender_contact JSONB NOT NULL,  -- {phone, whatsapp, sms, preferred}
   recipient_contact JSONB NOT NULL,  -- {phone, whatsapp, sms, preferred}
   preferred_language TEXT DEFAULT 'en',
   
   -- Visual Identification (future)
   visual_descriptors JSONB,  -- {clothing_color, landmarks}
   
   -- Payload
   payload_weight_declared FLOAT NOT NULL CHECK (payload_weight_declared <= 8.0),
   payload_weight_actual FLOAT,  -- After verification
   container_id TEXT,  -- Physical container tracking
   
   -- Mission Timeline
   status TEXT NOT NULL,  -- Current mission state
   reroute_count INTEGER DEFAULT 0,
   
   -- Energy
   estimated_energy_cost FLOAT,  -- Pre-flight
   actual_energy_used FLOAT,  -- Post-mission
   
   -- ETA Tracking
   eta_initial_seconds INTEGER,
   eta_actual_seconds INTEGER,
   
   -- Regulatory (NCAA Part 18)
   flight_authorization_id TEXT,  -- NCAA approval
   remote_pilot_id TEXT,  -- If operator intervened
   
   -- Timestamps
   mission_start_time TIMESTAMPTZ,
   mission_end_time TIMESTAMPTZ,
   created_at TIMESTAMPTZ DEFAULT NOW(),
   updated_at TIMESTAMPTZ DEFAULT NOW(),
   
   -- Termination
   termination_reason TEXT CHECK (termination_reason IN (
       'COMPLETE', 'ABORT_OPERATOR', 'ABORT_SYSTEM', 
       'EMERGENCY', 'TIMEOUT', 'CANCELLED'
   )),
   
   FOREIGN KEY (drone_id) REFERENCES drone_registry(drone_id),
   FOREIGN KEY (business_id) REFERENCES business_profiles(business_id)
);

-- Indexes
CREATE INDEX idx_missions_status ON missions (status);
CREATE INDEX idx_missions_business ON missions (business_id, mission_start_time);
CREATE INDEX idx_missions_geo_pickup ON missions USING GIST (pickup_waypoint);
CREATE INDEX idx_missions_geo_dest ON missions USING GIST (destination_waypoint);
	



6.2 Table: mission_state_transition (Append-only)
CREATE TABLE mission_state_transitions (
   id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
   mission_id UUID NOT NULL,
   from_state TEXT NOT NULL,
   to_state TEXT NOT NULL,
   trigger_reason TEXT,
   operator_id UUID NULL,
   timestamp TIMESTAMPTZ NOT NULL,
   drone_location GEOGRAPHY(POINT),
   battery_percent FLOAT,
   
   -- State Machine Validation (Mission State Machine Section 5)
   expected_next_states TEXT[],  -- Valid transitions from this state
   duration_in_previous_state_seconds INTEGER,  -- Timeout analysis
   state_machine_version TEXT DEFAULT '2.0',  -- Track evolution
   
   -- Dual-Path Logging (Autonomy Definition Section 12.1)
   local_command_used BOOLEAN DEFAULT FALSE,
   cloud_override BOOLEAN DEFAULT FALSE,
   
   -- Compliance
   FOREIGN KEY (mission_id) REFERENCES missions(mission_id)
);

-- Indexes
CREATE INDEX idx_transitions_mission ON mission_state_transitions (mission_id, timestamp);
CREATE INDEX idx_transitions_operator ON mission_state_transitions (operator_id) WHERE operator_id IS NOT NULL;

17.1 Graduated Compression Strategy
* After 30 days: downsample to 0.1Hz (10-second intervals)
* After 1 year: aggregate to minute-level statistics
* After 5 years: keep only anomalies and incidents


Implementation:
-- Create materialized views for aggregated telemetry
CREATE MATERIALIZED VIEW telemetry_10s
WITH (timescaledb.continuous) AS
SELECT time_bucket('10 seconds', timestamp) as bucket,
      drone_id,
      AVG(battery_percent) as battery_avg,
      MAX(battery_percent) as battery_max,
      MIN(battery_percent) as battery_min,
      AVG(ground_speed_mps) as speed_avg,
      -- Additional aggregates
FROM drone_telemetry
GROUP BY bucket, drone_id;
